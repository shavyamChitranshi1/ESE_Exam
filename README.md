# AI Smart Complaint Management System

## 1. Project Overview
A complete MERN stack application integrating OpenRouter AI to intelligently manage, categorize, and prioritize complaints. 

## 2. Features
- **Complaint Registration**: Submit complaints with text descriptions and locations.
- **Complaint Tracking**: Track real-time status of complaints (Pending, In Progress, Resolved).
- **AI-Based Categorization**: GPT-4o-mini powered analysis for automatic priority and department routing.
- **Secure Authentication**: JWT and bcrypt based secure login and signup.
- **Search & Filter**: Search complaints by location or category.
- **Dashboard**: Summary analytics of complaint statuses.
- **Auto-generated AI Response**: Professional automated acknowledgment messages generated by AI.

## 3. Tech Stack
- **Frontend**: React.js, Vite, Tailwind CSS, Axios, React Router DOM, React Toastify, Context API.
- **Backend**: Node.js, Express.js, MongoDB Atlas, Mongoose, JWT, bcryptjs, dotenv, cors.
- **AI Integration**: OpenRouter API (gpt-4o-mini).
- **Deployment**: Render

## 4. Installation Steps
1. Clone the repository
2. Navigate to the `backend` folder and run `npm install`
3. Navigate to the `frontend` folder and run `npm install`

## 5. Environment Variables
Create a `.env` file in the `backend` directory. See `.env.example` in the root for variables.

## 6. MongoDB Setup
Ensure your `MONGO_URI` is correctly pointing to your MongoDB Atlas cluster in the `.env` file.

## 7. Running Frontend
```bash
cd frontend
npm run dev
```

## 8. Running Backend
```bash
cd backend
npm run server # or node server.js
```

## 9. API Endpoints
- `POST /api/auth/signup` - Register a user
- `POST /api/auth/login` - Login user
- `GET /api/auth/me` - Get current user
- `GET /api/complaints` - Get all complaints
- `POST /api/complaints` - Create a new complaint
- `PUT /api/complaints/:id` - Update a complaint
- `DELETE /api/complaints/:id` - Delete a complaint
- `POST /api/ai/analyze` - Analyze complaint text with AI

## 10. Deployment Steps
This project is configured for Render via the `render.yaml` file.
1. Connect the repository to Render.
2. Render will automatically detect the Web Services for frontend and backend.
3. Configure the environment variables in the Render dashboard.
